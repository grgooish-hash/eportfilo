CSS

CSS (Cascading Style Sheets) separates how a page looks from what it contains. The same HTML can be restyled completely just by changing a stylesheet — this is the principle of "separation of concerns."

What this demonstrates

The demo below is a small dashboard of cards that shows five CSS techniques working together: Flexbox for the toolbar, CSS Grid for the responsive card layout, custom properties (variables) for theming, transitions for hover effects, and a keyframe animation. A single button toggles between light and dark themes by switching one attribute — every colour updates instantly because they're all driven by variables.

1. Custom properties for theming

Colours are declared once as variables on :root, then reused everywhere via var(--name). A second set of values is declared for [data-theme="dark"] — switching the attribute swaps every colour at once:

:root{
  --bg:#F7F5F0; --fg:#1A2332; --card:#fff; --accent:#E8A33D;
}
[data-theme="dark"]{
  --bg:#1A2332; --fg:#F7F5F0; --card:#2D3344;
}
body{ background:var(--bg); color:var(--fg); }

2. Responsive grid with auto-fit

The card grid doesn't use media queries to define column counts. Instead, repeat(auto-fit, minmax(190px, 1fr)) lets the browser fit as many 190px-minimum columns as will floor, stretching them to fill any remaining space:

.grid{
  display:grid;
  grid-template-columns: repeat(auto-fit, minmax(190px, 1fr));
  gap:1rem;
}

3. Transitions and keyframe animation

Hovering a card animates its position and shadow smoothly because transition tells the browser to interpolate property changes over time, rather than jumping instantly:

.card{
  transition: transform .25s ease, box-shadow .25s ease;
}
.card:hover{
  transform: translateY(-6px);
  box-shadow: 0 12px 24px -12px rgba(0,0,0,0.25);
}

The pulsing dot uses a @keyframes rule that scales and fades the element in a loop:

@keyframes pulse{
  0%, 100% { transform: scale(1);   opacity: 1;   }
  50%      { transform: scale(1.5); opacity: 0.4; }
}
.pulse{ animation: pulse 1.8s ease-in-out infinite; }

4. Respecting user preferences

Not everyone wants moving interfaces. A prefers-reduced-motion media query disables all transitions and animations for users who have requested this at the operating system level:

@media (prefers-reduced-motion: reduce){
  * { animation: none !important; transition: none !important; }
}

Why this matters

This demo shows that CSS is far more than colour and font choices — it handles layout (Flexbox, Grid), interaction feedback (transitions), motion (animations), theming (custom properties), and accessibility (reduced motion), all without a single line of JavaScript beyond the one-line theme toggle. This separation is what allows the Bootstrap framework, covered next, to provide an entire design system through CSS classes alone.
