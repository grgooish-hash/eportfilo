JavaScript

JavaScript is the only programming language that runs natively in every browser. It responds to user events, reads and updates the DOM (Document Object Model) in real time, and can communicate with servers in the background using fetch() — without reloading the page.

What this demonstrates

The demo below contains two parts. The first is a to-do list built entirely with JavaScript: adding, completing, and removing tasks all update the DOM dynamically. The second sends a real fetch() request to a public API and displays the JSON response, demonstrating asynchronous JavaScript with async/await and error handling.

    1. Selecting and creating elements

    JavaScript selects existing elements with document.getElementById, and creates new ones with document.createElement. Each task in the list is built as an <li> containing a <span> and a remove <button>, then appended to the list:

    const li = document.createElement('li');
const span = document.createElement('span');
span.textContent = task.text;

const delBtn = document.createElement('button');
delBtn.textContent = 'Remove';

li.append(span, delBtn);
taskList.appendChild(li);

    2. Handling events

    Event listeners connect user actions to functions. The example below adds a task either by clicking a button or pressing Enter — two different events triggering the same function:

    addBtn.addEventListener('click', addTask);

taskInput.addEventListener('keydown', (e) => {
  if (e.key === 'Enter') addTask();
});

    Clicking a task's text toggles a done property on its data object and re-renders the list — this is the core pattern of "state drives the DOM": JavaScript keeps the source of truth in a plain array (tasks), and the visible list is regenerated from it every time something changes.

    3. Asynchronous JavaScript with fetch

    The second part of the demo sends a network request and waits for the response using async/await, wrapped in a try...catch so the page doesn't break if the request fails:

    fetchBtn.addEventListener('click', async () => {
  fetchOutput.textContent = 'Loading…';
  try {
    const res = await fetch('https://dummyjson.com/quotes/random');
    const data = await res.json();
    fetchOutput.textContent = JSON.stringify(data, null, 2);
  } catch (err) {
    fetchOutput.textContent = 'Could not reach the network…';
  }
});

    This is exactly the mechanism described in Section 00's request lifecycle diagram: the browser sends a request, a server responds with JSON, and JavaScript updates the page with the result — no full page reload required.

    Bringing the stack together

    This portfolio itself demonstrates the stack in practice: semantic HTML provides the structure for every page, a shared CSS file (assets/css/style.css) styles all five sections consistently, the Bootstrap section shows how a framework accelerates common UI patterns, and JavaScript — as shown here — adds the interactivity that turns static content into a usable application.
