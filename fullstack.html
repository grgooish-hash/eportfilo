What is Full Stack Development?

A full stack developer works across every layer of a web application — from the markup and styling a user sees, through to the logic that runs in their browser, the server that processes requests, and the database that stores everything in between.

The two halves of a web app

Every dynamic website is really two applications talking to each other. The front end is everything that runs inside the user's browser: the HTML that gives the page its structure, the CSS that controls how it looks, and the JavaScript that makes it respond to clicks, typing, and other events. The back end runs on a server somewhere else entirely — it receives requests from the front end, talks to a database, applies business logic, and sends data back, usually as JSON.

A full stack developer is comfortable moving between both halves. They might spend the morning building a registration form in HTML and styling it with Bootstrap, then spend the afternoon writing the server-side code (in something like Node.js, PHP, or Python) that checks whether the email address already exists in the database before saving the new account.

The layers of this portfolio

This ePortfolio is organised as a stack, shown in the sidebar. Each layer builds on the one before it:

  • HTML — the skeleton. Defines what elements exist on the page and what they mean (a heading, a list, a form, a link).
  • CSS — the skin. Controls colour, spacing, layout, typography, and responsiveness without changing the underlying content.
  • Bootstrap — a CSS and JavaScript framework that provides a pre-built grid system, components, and utility classes, so common UI patterns (navbars, cards, modals, forms) can be assembled quickly and consistently.
  • JavaScript — the behaviour. Listens for events (clicks, form submissions, page loads) and updates the page dynamically without a full reload — this is what turns a static document into an interactive application.

Worked example: a request from click to response

To make this concrete, consider a "Sign Up" button on a website. The diagram below traces what happens across the stack when a user submits the form.

Diagram · request/response lifecycle

1. User fills in the Sign Up form

2. JavaScript validates the form

3. Browser sends request to the server

4. Server checks database

5. Server sends response back

6. Page updates without reloading

  1. The user fills in the HTML form and clicks Sign Up.
  2. JavaScript intercepts the click, runs client-side validation, and sends the data to the server using fetch().
  3. The server (back end) receives the request, checks the database for an existing account, and creates a new record.
  4. The server sends a JSON response back (success or error).
  5. JavaScript receives the response and updates the page — e.g. showing a "Welcome!" message — without reloading.
