Bootstrap Framework

Bootstrap is a front-end framework that provides a 12-column responsive grid, a large library of pre-styled components (navbars, cards, modals, forms, alerts), and JavaScript plugins for interactive widgets — all driven by simple HTML classes.

What this demonstrates

The demo below is a small project showcase page built using only Bootstrap 5 classes — no custom layout CSS was written. It combines a responsive navbar, a 12-column grid of cards, form controls, alerts, a progress bar, and a modal dialog triggered without writing any JavaScript.

1. The 12-column grid

Bootstrap's grid divides each row into 12 columns. col-md-4 means "take 4 of 12 columns (one third) on medium screens and up, but stack full-width on smaller screens" — this is how the three project cards become a single column on mobile automatically:

<div class="row g-3">
  <div class="col-12 col-md-4">...card 1...</div>
  <div class="col-12 col-md-4">...card 2...</div>
  <div class="col-12 col-md-4">...card 3...</div>
</div>

2. Components: navbar and cards

The navbar uses navbar-expand-lg so it collapses into a hamburger menu below the "large" breakpoint, with navbar-toggler handling the toggle. Each project is a card with a badge for its category and Bootstrap's button classes (btn btn-outline-dark):

<div class="card shadow-sm">
  <div class="card-body">
    <span class="badge text-bg-warning">Frontend</span>
    <h5 class="card-title">Weather App</h5>
    <p class="card-text small text-muted">...</p>
    <button class="btn btn-outline-dark btn-sm">View details</button>
  </div>
</div>

3. JavaScript plugins via data attributes

The "View details" buttons open a modal dialog. This requires Bootstrap's bundled JavaScript, but no custom script — the behaviour is wired up declaratively with data-bs-* attributes:

<button data-bs-toggle="modal" data-bs-target="#projectModal">
  View details
</button>

<div class="modal fade" id="projectModal">...</div>

4. Utility classes

Spacing, colour, and text utilities (mb-3, text-muted, shadow-sm, bg-dark) let small visual adjustments be made directly in markup, without writing additional CSS rules.

Bootstrap vs. hand-written CSS

The previous section showed CSS written from scratch — full control, but every layout and component had to be designed and coded. Bootstrap trades some of that control for speed and consistency: a responsive grid, accessible components, and common patterns are available immediately as classes. In practice, full stack developers often use Bootstrap (or a similar framework) for structure and components, then layer custom CSS — as in Section 02 — on top for branding.
