HTML

HTML (HyperText Markup Language) describes the structure and meaning of content — not how it looks. Semantic elements tell the browser, search engines, and assistive technologies what each part of a page is: a navigation menu, an article, a table of data, a form.

What this demonstrates

The example below is a single, self-contained HTML page for a recipe. It deliberately uses a wide range of semantic and structural elements rather than generic <div>s for everything, so that the document's structure is meaningful on its own — even before any CSS is applied.

Elements used, and why

  • <header> — groups the page title and introductory navigation.
  • <nav> — wraps the in-page links, with aria-label for screen readers.
  • <main> — the single block of primary content for the page.
  • <section> — groups related content (ingredients, method, nutrition), each with its own heading referenced via aria-labelledby.
  • <figure> / <figcaption> — associates an image with its caption as a single unit.
  • <table>, <caption>, <thead>, <th scope="row|col"> — a properly labelled data table, so a screen reader can announce "Calories: 410 kcal" correctly.
  • <details> / <summary> — a native, JavaScript-free disclosure widget for optional content.
  • <mark> — highlights text with semantic meaning (here, an optional ingredient).
  • <footer> — closes off the page with metadata.

Code excerpt

This snippet shows the nutrition table — note the use of scope attributes, which let assistive technology correctly associate each cell with its row and column heading:

<table>
  <caption>Approximate values</caption>
  <thead>
    <tr><th scope="col">Nutrient</th><th scope="col">Amount</th></tr>
  </thead>
  <tbody>
    <tr><th scope="row">Calories</th><td>410 kcal</td></tr>
    <tr><th scope="row">Protein</th><td>9 g</td></tr>
  </tbody>
</table>

Why semantics matter

Two pages can look identical once styled, but a page built from meaningful elements is easier to navigate with a keyboard or screen reader, ranks better in search engines, and is easier for other developers to read and maintain. Semantic HTML is the foundation the rest of the stack — CSS, Bootstrap, and JavaScript — builds on: CSS styles these elements, and JavaScript selects and manipulates them by their structure.
